## 重大 Bug 修复

### 问题
IODim 训练时 loss 不下降，recurrent weights 对输出影响太小 (0.21%)

### 根本原因
`glif3_brainstate.py` 中 `_current_factor` 被错误地除以了 `voltage_scale`：

```python
# 错误代码
self._current_factor = current_factor_mv / self.voltage_scale

# 正确代码
self._current_factor = (1.0 / self.C_m) * (1.0 - jnp.exp(-dt / self.tau_m)) * self.tau_m
```

原因：
- 原始 JAX 实现 (`sparse_layer.py`)：权重在 `prepare_recurrent_connectivity` 中除以 `voltage_scale`
- Brainstate 实现 (`connectivity_brainstate.py`)：权重**不变**
- 但 `glif3_brainstate.py` 却对 `_current_factor` 除以了 `voltage_scale`
- 导致 recurrent contribution 被缩小 ~22 倍

### 修复效果

| 指标 | 修复前 | 修复后 |
|-----|-------|-------|
| _current_factor | 0.00025 | 0.0053 (21x) |
| Recurrent contribution | 0.21% | 4.45% (21x) |
| grad_norm | 8e-4 | 1.4e-1 (175x) |
| **Loss reduction** | -0.6% | **10.7%** ✅ |

- `src/v1_jax/nn/glif3_brainstate.py` - 第 166-172 行
