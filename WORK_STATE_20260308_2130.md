## 关键发现

### 1. IODim Loss 不下降问题 - 已解决

**根本原因**：Recurrent weights 对网络输出的影响太小

| 网络类型 | Recurrent contribution | 阈值贡献比例 |
|---------|----------------------|-------------|
| 小测试网络 (100n) | 0.024 voltage | 2.44% |
| Billeh 网络 (52K) | 0.002 voltage | 0.21% |

**分析**：
- 外部输入电流 ~820 才能引起 spike
- Recurrent PSC 贡献只有 ~0.02-0.002 normalized voltage
- **权重变化几乎不改变网络输出**

### 2. 修复方案验证

**小网络修复（已验证成功）**：
- 使用更大的初始权重 (50x instead of 5x)
- 短暂刺激后让 recurrent dynamics 主导
- 只在 post-stimulus 期间计算 loss
- **结果：Loss 从 0.099 降到 0.070 (29.4% reduction)**

**Billeh 网络**：
- 设计上是 LGN 驱动的，recurrent 只起辅助作用
- 正确的训练方式应该是：
  1. 训练 readout layer (V1 → 分类)
  2. 或者训练 input layer (LGN → V1)
  3. 不是训练 recurrent weights

## 技术细节

### Billeh 网络参数
- neurons: 51,978
- synapses: 3,574,968
- avg weight: 2.99
- avg in-degree: 68.8
- _current_factor: ~0.00025
- _psc_initial: ~0.49

### PSC → Voltage 计算
```
syn_input = weight * spike_count
psc_rise = psc_initial * syn_input
psc = psc_rise * syn_decay  (after 1 step)
voltage_contribution = psc * current_factor
```

### 为什么 Billeh 网络的 recurrent 影响小
1. 权重小 (mean=3 vs 需要 ~100)
2. current_factor 小 (~0.00025)
3. 设计目的是 LGN 驱动，recurrent 只做调制
